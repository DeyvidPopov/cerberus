# Login band-threshold tuning

Generated by `npm run eval:tune --workspace @cerberus/server` (ADR-0014).
Reproducible: fixed seed `20240601`.

Sweeps the **production behavioral score** (Mahalanobis → χ² CDF, the value
`scoreSample` returns at login) on a VALIDATION split that is DISJOINT from the
K&M test set used for the reported keystroke EER (**no tuning-on-test**, PROJECT.md §6):

- baseline fit on reps `[0, 150)`;
- genuine validation on reps `[150, 200)`;
- impostor validation on other subjects' reps `[5, 10)`
  (disjoint from the K&M impostor test, reps `[0, 5)`);
- the K&M genuine test set, reps `[200, end)`, is **never read** here.

- **Subjects:** 51 · **Validation genuine:** 2550 · **Validation impostor:** 12750
- **Behavioral combiner weight:** 0.5 (composite when only behavioral fires = weight · score).

The keystroke χ² score is a **soft** login signal: genuine validation scores cluster
near 0 (good fit) while impostors spread high. Its equal-error point sits at a tiny
score threshold (≈ 0.002, EER **19.25%**) — but operating there
would step up **19.25%** of genuine logins, which is poor UX. So step-up is set to
the most sensitive composite threshold that keeps the genuine false-step-up rate within
budget; the residual behavioral FAR is closed by **contextual stacking + the TOTP
step-up** (ADR-0012), not by the behavioral signal alone.

|------------------:|-------------------:|--------------------:|----------------------:|

| 0.20 | 0.40 | 45.22% | 8.20% |
| 0.25 | 0.50 | 47.38% | 7.22% |

| 0.40 | 0.80 | 53.37% | 5.84% |
| 0.50 | 1.00 | 100.00% | 0.00% |

## Chosen operating point

- **stepUp = 0.29** — the most sensitive composite threshold whose
  genuine false-step-up rate stays ≤ 7.00% on validation (here FRR 6.98%,
  behavioral-only FAR 48.84%). Lower thresholds cut FAR only marginally while
  multiplying genuine step-ups; the combiner + context carry the rest.
- **deny = 0.70** — kept above the maximum single-signal contribution
  (max weight = 0.5), so a deny requires STACKED signals (ADR-0012). No single
  behavioral or contextual signal can deny alone.

_The live values are the named config in `apps/server/src/risk/config.ts`
(DEFAULT_BAND_THRESHOLDS / DEFAULT_COMBINER_WEIGHTS), set with this rationale._
